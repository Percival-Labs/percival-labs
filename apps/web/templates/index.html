The Problem

The Wild West of AI Skills

AI skill registries today have zero verification. You download a skill, trust it blindly, and hope for the best. That is not engineering -- that is negligence.

Zero Verification

Most registries accept anything. No manifest checks, no static analysis, no security scanning. Ship it and pray.

Supply Chain Attacks

Skills can exfiltrate data, escalate privileges, or inject instructions. No one is checking what skills actually do.

Blind Trust

No trust scoring, no quality metrics, no usage data. Every skill looks the same whether it is battle-tested or brand new.

Capabilities

Built for Security-Conscious Teams

Every feature designed around the principle that trust must be earned, verified, and continuously validated.

Security Verification

Two-layer verification: L1 manifest validation plus L2 static analysis with secret detection and dangerous pattern scanning.

Trust Scoring

percival CLI

The command-line interface for interacting with the Percival Labs registry. Search skills, verify security, and install with confidence.

  • Search across 91+ verified skills with full-text matching
  • Run L1 + L2 security verification on any skill file
  • Install skills directly into your project
  • Publish and manage your own skills
  • JSON output for CI/CD pipeline integration
percival-cli
# Search the registry $ percival search --category security Found 12 skills in "security" Security L2 verified trust: 0.94 OSINT L2 verified trust: 0.91 FootprintScan L1 verified trust: 0.87
# Verify a local skill $ percival verify ./skills/MySkill/SKILL.md L1 Manifest PASS schema valid, 3 capabilities L2 Analysis PASS 0 secrets, 0 dangerous patterns Trust Score 0.92 ready to publish
# Install a verified skill $ percival install @percival/security Installed @percival/security v1.2.0
